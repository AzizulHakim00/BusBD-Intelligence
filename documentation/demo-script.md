# Demonstration flow

1. Open the home page and search **Dhaka → Chattogram**.
2. Select a trip, choose a seat and complete mock payment.
3. Save the `BBD...` reference and open **My trips** to retrieve the QR ticket.
4. Open **Live tracking** and wait for the GPS simulator to publish a location.
5. Submit a support complaint.
6. Open **Operations** and sign in as `admin@busbd.local` / `Admin123!`.
7. Show fleet health, booking revenue and the AI-classified support queue.
8. Open `/swagger-ui.html` to demonstrate the REST contract.
9. For the full local topology, run `docker compose up --build`.
